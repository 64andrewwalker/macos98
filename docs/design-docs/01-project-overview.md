# 01 - 项目概述

## 项目定位

> 一个运行在浏览器中的「伪桌面操作系统」，本质是一个复杂的单页应用（SPA），模拟 Classic Mac OS (System 7) 的视觉风格与交互体验。
> 

### 这个项目 **是**

- 一个前端架构实验项目
- 一个复杂状态管理的学习载体
- 一个 Classic Mac OS 的怀旧致敬
- 一个可在浏览器离线运行的演示系统

### 这个项目 **不是**

- 真正的操作系统内核
- 具有安全隔离的多用户系统
- 适合长期生产工作的数据存储方案
- 可跨设备同步的云端应用

---

## 核心约束

<aside>
⚠️

**浏览器平台的固有限制**

1. **单线程模型** - JavaScript 主线程阻塞会导致 UI 失去响应
2. **无真正进程隔离** - 所有代码共享同一个 `window` 上下文
3. **存储不可靠** - 用户清缓存/换浏览器，数据全部丢失
4. **配额有限** - 每个 origin 的存储空间受浏览器限制
</aside>

---

## 技术选型

| 领域 | 选型 | 理由 |
| --- | --- | --- |
| **UI 框架** | React / Vue 3 | 组件化、生态成熟 |
| **状态管理** | Zustand / Pinia | 轻量、TypeScript 友好 |
| **持久化存储** | IndexedDB (via idb) | 结构化数据、大容量 |
| **样式隔离** | Shadow DOM | 原生 CSS 隔离 |
| **类型系统** | TypeScript | 类型安全、重构友好 |
| **测试框架** | Vitest + Testing Library | 快速、现代 |

## 成功标准 (MVP)

- [ ]  能启动系统，显示桌面和菜单栏
- [ ]  能打开 Finder 浏览虚拟文件系统
- [ ]  能打开 TextEdit 创建/编辑/保存文本文件
- [ ]  刷新页面后，文件仍然存在
- [ ]  窗口可拖拽、缩放、关闭、切换焦点
